# 강의 요약본(1~2페이지용)

## 1. 교육 목표
- SSI 핵심 개념(DID/VC/VP)을 이해하고, 간단한 데모 시스템으로 실습한다.
- gRPC 기반 분산 구성과 역할 분리(Registry/Registrar/Issuer/RP/Holder)를 체험한다.

## 2. 시스템 구성
- DID-Registry: DID Document 저장소(LevelDB)
- DID-Registrar: DID method 기반 라우팅 및 Resolver
- demo-issuer: VC 발급 기관
- demo-rp: 인증 챌린지 및 VP 검증
- demo-client: 시나리오 실행(Holder 역할)
- did_service_endpoint: REST 데모 + Swagger 문서

## 3. 핵심 흐름
1) DID 생성/등록/해결
2) 인증 챌린지/응답
3) VC 발급
4) VP 제출/검증
5) (확장) 신원 VC → 계약 VC → 접근 제어 시나리오

## 4. 프로토콜/형태
- gRPC: 내부 서비스 통신(학습 핵심)
- JWT: VC/VP 서명 포맷
- REST: 보조 채널(문서화/테스트, Android 데모 연동)

## 5. 코어 재사용 구조 요약
- 동일한 Go 코어 로직을 기준으로 gRPC/REST/web 서버와 c-shared 라이브러리 형태로 재사용 가능
- 터미널 데모(Go), REST API(did_service_endpoint), Android/JNI, IoT 디바이스까지 동일 코어 기반 확장
- 디바이스별 ABI(armv7/arm64/x86 등) 빌드와 배포 방식은 다르므로 환경별 빌드가 필요
- 현재 데모는 REST 기반 흐름이 가장 완성도가 높으며 gRPC는 병행 확장 가능

## 6. 실습 준비
- 로컬 실행: `make dev-up`
- 포트: 50051~50055
- 필수 환경변수: `ETH_PRIVATE_KEY_HEX`

## 7. 학습 결과
- DID 기반 인증 흐름 이해
- VC/VP 검증 체험
- 서비스 역할 분리 및 상호 연동 이해
