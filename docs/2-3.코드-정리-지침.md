# 코드 정리 지침

작업을 수행할 때 **수정하는 코드와 주변(Context)을 더 매끄럽게 정리**합니다. (Boy Scout Rule)

## ✅ 정리 대상 체크리스트
1. **이름(Naming)**: 모호한 이름(`data`, `flag`)은 의도가 드러나는 이름(`userProfile`, `isLoaded`)으로 변경.
2. **로그(Logging)**: 단순 `console.log` 삭제보다는 **디버깅 레벨(`console.debug`) 적용 또는 `console.group`으로 구조화**하여 의미 있는 로그 자산으로 남김. (잡음/노이즈성 로그는 삭제)
3. **정리(Clean-up)**: 죽은 코드, 불필요한 주석 제거.
4. **타이핑(Typing)**: `any`는 구체적 타입으로 명시하되, **타입 정의에 10분 이상 소요되면 생략**한다.
5. **구조(Structure)**: 중첩된 `if/else`는 **Guard Clause(조기 반환)** 패턴으로 평탄화.
6. **상수(Constants)**: 문자열 리터럴, 매직 넘버 등 의미를 알 수 없는 값은 **상수로 추출**해 관리한다.

## ⛔️ 주의사항
1. **동작 변경 금지(Behavior Preservation)**  
   - Polishing은 로직을 바꾸는 것이 아니라 품질/가독성을 개선하는 작업이다.  
   - **“이 기능이 있으면 좋겠다”는 생각이 들어도 과제 범위 밖이면 추가 금지.**
2. **과몰입 금지**: 전체 작업 시간의 **20%**를 넘기지 않는다.
3. **보고**: PR/요약 시 **[Polish]** 항목으로 적용 내역을 분리해 기록한다.
