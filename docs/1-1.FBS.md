# 기능분해구조(FBS)

## 1. 문서 목적
- 교육/실습 관점에서 기능을 계층적으로 분해하여 전체 기능 범위를 한눈에 파악한다.
- 개발/테스트 범위를 기능 단위로 설명할 수 있게 한다.

## 2. 기능 분해
### 2.1 DID 관리
- DID 생성
- DID 등록(Registry 저장)
- DID 해석(Resolve)
- DID 업데이트

### 2.2 DID 인증(Simple Presentation)
- 챌린지 발급
- 챌린지 응답 생성
- 응답 검증(시간 제한 포함)

### 2.3 VC 발급/검증
- VC 발급(서명)
- VC 검증(서명/만료)
- 신원 VC 발급(기관)
- 계약 VC 발급(민간)

### 2.4 VP 생성/검증
- VP 생성(VC 포함)
- VP 생성(Simple Presentation: VC 없음)
- VP 검증(서명/클레임)
- VP 검증(aud/nonce 요구 사항)

### 2.5 REST 데모 API
- DID API: create/get/public-key
- VC/VP API: create/verify
- 데모 플로우 API:
  - demo actors 조회
  - license challenge/issue
  - rental challenge/issue

### 2.6 Android 데모 앱
- DID 생성
- 신원 VC 발급(기관)
- 계약 VC 발급(민간)
- 차량 접근 제어(VP 제출/검증)
- 만료 시간 입력/카운트다운 표시
- 체크리스트 UI(검증 단계 표시)

### 2.7 운영/보조 기능
- 로컬 실행 스크립트(`make dev-up`)
- 로그 수집(`.devlogs/`)
- API 문서화(Swagger/Redocly)
- 테스트/커버리지 실행
